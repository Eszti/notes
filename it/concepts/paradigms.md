# Paradigms

## Imperative

- programmer instructions
- types
  - procedural
  - object-oriented

## Declarative

- declare desired result
- types
  - functional
  	- applying and composing functions
  	- functions: first-class citizens
  	- e.g.: Erlang, Haskell 
  - logic
  	- based on formal logic
  	- e.g.: Prolog
  - mathematical

## AOP 

- Aspect-oriented Programming
- adding additional behaviour to existing code (without modifying the code)
- ~ annotations
